Introduction¶

This report explores deeper the '2014 Mental Health in Tech' survey data, which uses self-reported responses from individuals from various backgrounds. Its goals is to identify and highlight the mental health of individuals that may need help, here I created a predictive model that determines whether or not a survey taker has sought mental health treatment.

The analysis is conducted using the following tools and libraries:

  • Python 3.12 for development and scripting (Any version that supports these libraries should work fine)

  • Pandas for data manipulation and preprocessing

  • NumPy for numerical operations

  • Matplotlib & Seaborn for visualizations

  • Scikit-learn for model building, transformation pipelines, and evaluation

The workflow follows these major steps:

  • Data Cleaning: Handling inconsistent gender values, outlier ages, and missing data

  • Feature Engineering: Binning age groups and one-hot encoding categorical variables

  • Model Construction: Using a pipeline to preprocess features and train a RandomForestClassifier

  • Bias Analysis: Evaluating feature importance and subgroup-level accuracy (e.g., gender, age)

  • Reflection: Interpreting how bias in the dataset influences predictions

def normalize_gender(g):
    g = str(g).strip().lower()
    if g in ['male', 'm', 'cis male', 'man']:
        return 'Male'
    elif g in ['female', 'f', 'cis female', 'woman', 'female (cis)']:
        return 'Female'
    else:
        return 'Nonbinary/Other'

# Normalize Gender
df_clean['normalized_gender'] = df_clean['Gender'].apply(normalize_gender)

# Normalize Age
df_clean = df_clean[(df_clean['Age'] > 17) & (df_clean['Age'] <= 80)]

# Build transformer and pipeline
preprocessor = ColumnTransformer(
    transformers=[
        ('cat', OneHotEncoder(drop='first', handle_unknown='ignore'), categorical_cols)
    ],
    remainder='passthrough'
)

pipeline = Pipeline(steps=[
    ('preprocessor', preprocessor),
    ('classifier', RandomForestClassifier(random_state=42))
])

# Fit model
pipeline.fit(X_train, y_train)

Implications of Bias¶

  • Age is the most influential feature, which may reflect genuine trends in treatment-seeking behavior — but it can also overfit to age-specific biases in access or perception of care.

  • Gender and Country Effects:

    • normalized_gender_Male shows the model behaves differently based on gender.
    • Country_United States and other country features (e.g., UK, Canada) rank high, meaning the model’s decision process is heavily skewed toward regional patterns.
    • This is expected due to overrepresentation of U.S. and English-speaking countries in the dataset.

  • Employment Status (self_employed_Yes) appears in the top features, potentially reflecting systemic differences in insurance coverage or workplace benefits that correlate with mental health access.

Fairness Reflection¶

These findings underscore that the model may not be learning why someone seeks treatment — but rather who is most likely to based on historically biased factors.

This may not be an issue for exploratory research, but if deployed in real-world scenarios, it could lead to discriminatory outcomes if predictions are used to allocate resources or interventions.

To summarize: Biased data leads to biased models. The dominance of gender and nationality in feature importance confirms the model is encoding demographic disparities rather than universal indicators of mental health need.

from sklearn.metrics import accuracy_score

# Make predictions
y_pred = pipeline.predict(X_test)

# Copy test set to match with predictions
X_test_copy = X_test.copy()
X_test_copy['y_true'] = y_test
X_test_copy['y_pred'] = y_pred

# Restore demographic columns (normalized_gender + Age) from full df_clean
X_test_copy['normalized_gender'] = df_clean.loc[X_test_copy.index, 'normalized_gender']
X_test_copy['Age'] = df_clean.loc[X_test_copy.index, 'Age']

# Create age bins
bins = [18, 25, 35, 45, 60, 80]
labels = ['18–24', '25–34', '35–44', '45–59', '60+']
X_test_copy['age_group'] = pd.cut(X_test_copy['Age'], bins=bins, labels=labels, right=False)

# Accuracy by gender
gender_acc = X_test_copy.groupby('normalized_gender').apply(
    lambda g: accuracy_score(g['y_true'], g['y_pred'])
).reset_index(name='Accuracy')

# Accuracy by age group
age_acc = X_test_copy.groupby('age_group').apply(
    lambda g: accuracy_score(g['y_true'], g['y_pred'])
).reset_index(name='Accuracy')

print("Accuracy by Gender:")
print(gender_acc)

print("\nAccuracy by Age Group:")
print(age_acc)

These results further reinforce that model performance is not uniform across demographic lines — an important observation when analyzing how biased datasets can yield discriminatory behavior in predictive systems.

Conclusion¶

Conclusion & Reflection on Bias in Predictive Modeling¶

This study set out with the hypothesis that demographic and cultural biases embedded in the '2014 Mental Health in Tech' survey dataset would manifest in machine learning predictions. The results confirm this.

Evidence from the Model¶

  • Top features influencing prediction included:

    • Age
    • normalized_gender_Male
    • Country_United States
    • self_employed_Yes

  • Subgroup accuracy showed clear disparities:

    • Women were predicted more accurately than men or nonbinary respondents.
    • Older adults (60+) had perfect prediction scores, while the 35–44 age group had the lowest accuracy.
    • Many countries, particularly those with fewer samples, had inflated or unreliable accuracy.

These outcomes align with the original concern: biased training data leads to biased models. The model reflects the overrepresentation of U.S.-based, male, and younger respondents. Rather than generalizing fairly, the model amplifies those patterns.

Ethical Implications¶

If this model were deployed in the real world — for example, as part of an HR mental health prediction tool — it could:

  • Misclassify individuals from underrepresented countries or identities.
  • Amplify existing cultural stigmas about mental health.
  • Undermine trust in AI-based recommendations due to uneven treatment across groups.

Final Thoughts¶

This report demonstrates that machine learning models can produce seemingly high-performing yet fundamentally skewed results. This analysis demonstrates the risks of encoding social and cultural disparities into machine learning predictions, especially in sensitive domains like mental health. Further work should explore strategies for mitigating such biases while preserving the predictive power of behavioral insights.
